Minishell: test

As beautiful as a shell can be.

01 // What It Does

This was my first major systems project at 42 Network. The goal was simple but terrifying: recreate a functioning shell (like Bash) from scratch using C.

It handles command execution, pipes (|), redirections (< > >>), environment variables, and signals (ctrl-C, ctrl-D). It basically allows you to interact with the kernel just like your standard terminal.

02 // The Learning Motivation

Before this, "the terminal" was a black box to me. I typed ls -la and magic happened.

I built this to demystify that process. I wanted to understand exactly how the OS creates new processes (fork), how programs talk to each other (pipe), and how file descriptors actually work. You can't exploit what you don't understand.

03 // Challenges Faced

The Parsing Nightmare

Handling quotes was harder than the execution itself. Making sure that echo "hello world" preserves spaces but echo hello world doesn't required writing a custom lexer.

Memory Leaks

Since 42 allows zero memory leaks, I had to manually manage every malloc. I spent days with Valgrind tracking down 4 bytes that weren't being freed when a process crashed.

if (pipe(pipefd) == -1)
    return (perror("pipe"), 1);
pid = fork();
if (pid == 0) // Child Process
{
    close(pipefd[0]); // Close read end
    dup2(pipefd[1], STDOUT_FILENO); // Redirect stdout to pipe
    execve(path, args, env);
}

04 // Key Takeaways

  • Process Management: Deep understanding of the process lifecycle (fork -> execve -> waitpid).
  • File Descriptors: How standard input/output are just integers pointing to streams.
  • Signals: How to intercept SIGINT so my shell doesn't close when I press Ctrl-C.
